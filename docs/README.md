## 二、项目完整性概览

- **阶段一～五、阶段六（大部分）、阶段七**：已完成。具备登录注册、用户/装备/计划/推荐/收藏/加购/评分、管理端用户与装备与角色审核、计划需求描述与关键词、用户画像与难度偏好、活动类型→标签映射与冷启动兜底、联调文档、单元测试、部署说明。
- **未完成或可选**：社区论坛、独立购物车页面、个人中心扩展（资料编辑、收藏/订单列表、积分等级）、内容审核与推荐配置、性能压测与 Docker 生产镜像等（见毕设实施计划中的勾选与「前端-待补」）。
- **文档**：以本索引与《毕设实施计划》的「当前进度」「项目进度检查」为准；《快速开始》为运行入口；《推荐算法说明》+《推荐逻辑优化方案》覆盖推荐设计与使用，避免重复维护多份「推荐说明」。

---

## 三、阅读顺序建议

1. **只想把项目跑起来**： [快速开始.md](快速开始.md) → 按清单执行迁移（若有）→ 启动后端与前端。
2. **做联调与测试**： [阶段七-联调与测试.md](阶段七-联调与测试.md) → [快速开始.md](快速开始.md) 常用命令。
3. **写论文或答辩**： [毕设实施计划.md](毕设实施计划.md) + [推荐算法说明.md](推荐算法说明.md) + [阶段二-数据库设计.md](阶段二-数据库设计.md) + [部署说明.md](部署说明.md)。
4. **排查推荐效果**： [推荐逻辑优化方案.md](推荐逻辑优化方案.md) 自检与已实现优化。
5. **查找代码文件**： [项目文件清单.md](项目文件清单.md)。
