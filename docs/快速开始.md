# 户外运动装备推荐系统 - 快速开始

本项目后端基于 **Spring Boot 4 + JPA + MySQL + Redis**，开发环境已按开题报告配置。下面是本机快速运行步骤。

---

## 1. 环境准备（一次性）

- **JDK 17+**
- **Maven 3.9+**
- **Docker Desktop**（已启用 WSL2）

> 说明：MySQL 与 Redis 通过 Docker 容器运行，无需在 Windows 上单独安装服务。

---

## 2. 启动依赖服务（MySQL + Redis）

在项目根目录 `D:\outdoor-gear-recommendation` 执行：

```bash
cd /d D:\outdoor-gear-recommendation
docker compose up -d
```

- 会启动：
  - `outdoor-gear-mysql`：MySQL 8，库名 `outdoor_gear`，账号 `outdoor_gear/outdoor_gear_dev`
  - `outdoor-gear-redis`：Redis 7，端口 6379
- 通过 `docker ps` 可查看容器状态。

首次启动后已在容器内执行建表脚本 `src/main/resources/sql/schema.sql`，包含用户、角色、装备、出行计划等全部表结构。

---

## 3. 启动后端服务

本地开发使用 `local` 配置，连接 Docker 内的 MySQL：

```bash
cd /d D:\outdoor-gear-recommendation
mvn spring-boot:run -Dspring-boot.run.profiles=local
```

启动成功日志中可见：

- `Started OutdoorGearRecommendationApplication ...`
- 活动 profile：`local`
- Tomcat 端口：`8080`

如需在未启动 MySQL/Redis 的情况下仅启动 Web 层，可使用 `no-db` 配置：

```bash
mvn spring-boot:run -Dspring-boot.run.profiles=no-db
```

**本地配置 `application-local.yml`**（`src/main/resources/application-local.yml`）：

| 配置项 | 说明 | 默认 |
|--------|------|------|
| `gear.init.force` | 是否强制重新初始化装备知识库（分类、标签、装备及变体）。为 `true` 时会清空并重新写入，适用于需要刷新装备数据或修复数据问题的场景；为 `false` 时若库中已有数据则跳过初始化。 | `false` |

首次启动或希望重新生成装备数据时，可将 `gear.init.force` 设为 `true`，启动一次后再改回 `false`，避免每次启动都全量初始化。

## 4. 验证运行是否正常

项目中提供了若干测试接口，用于验证 JPA 与数据库连接。均为 `GET` 请求，在浏览器访问即可。

| 接口 | URL | 说明 |
|------|-----|------|
| 用户列表 | `http://localhost:8080/api/test/users` | 返回所有用户（含 admin） |
| 用户行为 | `http://localhost:8080/api/test/behavior` | 创建/返回收藏与评分记录 |
| 订单与推荐 | `http://localhost:8080/api/test/order` | 创建/返回测试订单与推荐结果 |
| 装备分类 | `http://localhost:8080/api/test/gears/categories` | 返回装备分类（启动时自动初始化 20 类） |
| 装备列表 | `http://localhost:8080/api/test/gears/items` | 返回装备列表（含 1000+ 装备及变体、标签） |
| 装备汇总 | `http://localhost:8080/api/test/gears` | 返回分类、装备、标签汇总 |
| 出行计划 | `http://localhost:8080/api/test/plans` | 创建/返回用户 1 的出行计划及关联装备 |
| 登录 | `POST /api/auth/login` | 请求体 `{"username":"admin","password":"admin123"}`，返回 JWT |
| 注册 | `POST /api/auth/register` | 请求体 `{"username":"xxx","password":"xxx"}` |
| 个人信息 | `GET /api/user/me` | 需 Header `Authorization: Bearer <token>` |
| 修改资料 | `PUT /api/user/me` | 请求体 `{"nickname":"xxx","phone":"","email":""}` |
| 角色申请 | `POST /api/user/role-apply` | 请求体 `{"roleCode":"ROLE_EXPERT","reason":"xxx"}`（仅专家可申请） |
| 管理员-用户列表 | `GET /api/admin/users` | 需 ADMIN 角色 |
| 管理员-角色列表 | `GET /api/admin/roles` | 需 ADMIN 角色 |
| 管理员-审核申请 | `PUT /api/admin/role-apply/{id}/review` | 请求体 `{"status":1,"adminNote":"xxx"}` |
| **装备分类** | `GET /api/gears/categories` | 公开，无需登录 |
| **装备列表** | `GET /api/gears?categoryId=&name=&page=0&size=20` | 公开，支持分页筛选 |
| **装备详情** | `GET /api/gears/{id}` | 公开，登录用户会记录浏览 |
| **装备对比** | `GET /api/gears/compare?ids=1,2,3` | 需登录 |
| **装备评分** | `POST /api/gears/{id}/rating` | 请求体 `{"score":5,"comment":"xxx"}`，需登录 |
| **收藏** | `POST /api/user/favorites/{gearId}` | 需登录 |
| **取消收藏** | `DELETE /api/user/favorites/{gearId}` | 需登录 |
| **加购** | `POST /api/user/cart` | 请求体 `{"gearId":1,"quantity":1}`，需登录 |
| **购物车列表** | `GET /api/user/cart` | 需登录 |
| **出行计划列表** | `GET /api/user/plans` | 需登录 |
| **出行计划详情** | `GET /api/user/plans/{id}` | 需登录 |
| **创建计划** | `POST /api/user/plans` | 请求体含 name、destination、season 等，需登录 |
| **编辑计划** | `PUT /api/user/plans/{id}` | 需登录 |
| **删除计划** | `DELETE /api/user/plans/{id}` | 需登录 |
| **计划添加装备** | `POST /api/user/plans/{id}/gears` | 请求体 `{"gearId":1,"quantity":1}`，需登录 |
| **计划移除装备** | `DELETE /api/user/plans/{planId}/gears/{gearId}` | 需登录 |
| **管理员-装备列表** | `GET /api/admin/gears` | 需 ADMIN/GEAR_ADMIN/EXPERT |
| **管理员-装备 CRUD** | `POST/PUT/DELETE /api/admin/gears` | 需 ADMIN/GEAR_ADMIN/EXPERT |
| **按计划推荐** | `GET /api/recommend/by-plan?planId=1&topN=20` | 需登录且为计划所属用户 |
| **按约束推荐** | `POST /api/recommend` | 请求体 `{"planId":1}` 或 `{"userId":1,"constraint":{...}}` |

**说明**：
- 若数据库已存在，需执行 `sql/migration-role-apply.sql` 添加角色申请表。
- 若需用户偏好（冷启动优化），执行 `sql/migration-user-preference.sql` 添加用户偏好表。
- **计划关键词与用户画像**：执行 `sql/migration-plan-keywords-and-user-profile.sql` 添加计划需求描述、提取关键词及用户偏好扩展字段（否则启动会因表结构不匹配报错）。
- `local` profile 会排除 Redis，推荐接口仍可正常使用，但无缓存；启用 Redis 时推荐结果会缓存 30 分钟。

预期行为：

- 首次启动 `local` profile 时，会自动插入：
  - 角色：`ROLE_ADMIN`、`ROLE_USER`
  - 用户：`admin`（密码已加密存储）
  - 装备知识库：20 个分类、120+ 标签、1000+ 装备及变体（含装备-标签关联）
- 调用 `/api/test/users` 应返回 JSON 数组，其中包含 `admin` 用户记录。
- 调用 `/api/test/gears` 可查看分类、装备、标签的完整知识库。

如果看到以上返回，说明：

- Docker 中的 MySQL/Redis 工作正常；
- Spring Boot 数据源配置正确；
- JPA 实体与仓库读写无误。

## 5. 常用命令汇总

```bash
# 启动依赖服务（MySQL + Redis）
docker compose up -d

# 查看容器状态
docker ps

# 停止依赖服务
docker compose down

# 启动后端（local 配置，连 Docker MySQL/Redis）
mvn spring-boot:run -Dspring-boot.run.profiles=local

# 仅启用 Web（不连数据库/Redis）
mvn spring-boot:run -Dspring-boot.run.profiles=no-db
```

---

## 6. 启动前端（Vue 3 + Element Plus）

```bash
cd frontend
npm install
npm run dev
```

前端默认运行在 `http://localhost:5173`，Vite 会将 `/api` 请求代理到后端 `http://localhost:8080`。需先启动后端（步骤 3）。

---

## 7. 联调验证登录流程

1. **启动顺序**：先启动 Docker（MySQL + Redis）→ 后端 → 前端

2. **打开前端**：浏览器访问 `http://localhost:5173`

3. **登录验证**：
   - 点击右上角「登录」
   - 输入测试账号：`admin` / `admin123`
   - 点击「登录」后应跳转首页，右上角显示用户名
   - 点击用户名下拉可退出

4. **注册验证**：
   - 点击「注册」→ 输入新用户名、密码、确认密码
   - 注册成功后自动登录并跳转首页

5. **Token 验证**：
   - 登录后刷新页面，应保持登录状态（Token 从 localStorage 恢复）
   - 打开开发者工具 → Application → Local Storage → 可看到 `token`、`user`

6. **常见问题**：
   - 后端未启动：登录会报网络错误，检查 `http://localhost:8080` 是否可访问
   - 401 未授权：Token 过期或无效时会自动跳转登录页
   - 代理不生效：确认前端通过 `npm run dev` 启动（Vite 代理仅开发环境生效）
