# 户外运动装备推荐系统 - 快速开始

本项目后端基于 **Spring Boot 4 + JPA + MySQL + Redis**，开发环境已按开题报告配置。下面是本机快速运行步骤。

---

## 1. 环境准备（一次性）

- **JDK 17+**
- **Maven 3.9+**
- **Docker Desktop**（已启用 WSL2）

> 说明：MySQL 与 Redis 通过 Docker 容器运行，无需在 Windows 上单独安装服务。

---

## 2. 启动依赖服务（MySQL + Redis）

在项目根目录 `D:\outdoor-gear-recommendation` 执行：

```bash
cd /d D:\outdoor-gear-recommendation
docker compose up -d
```

- 会启动：
  - `outdoor-gear-mysql`：MySQL 8，库名 `outdoor_gear`，账号 `outdoor_gear/outdoor_gear_dev`
  - `outdoor-gear-redis`：Redis 7，端口 6379
- 通过 `docker ps` 可查看容器状态。

首次启动后已在容器内执行建表脚本 `src/main/resources/sql/schema.sql`，包含用户、角色、装备、出行计划等全部表结构。

---

## 3. 启动后端服务

本地开发使用 `local` 配置，连接 Docker 内的 MySQL：

```bash
cd /d D:\outdoor-gear-recommendation
mvn spring-boot:run -Dspring-boot.run.profiles=local
```

启动成功日志中可见：

- `Started OutdoorGearRecommendationApplication ...`
- 活动 profile：`local`
- Tomcat 端口：`8080`

如需在未启动 MySQL/Redis 的情况下仅启动 Web 层，可使用 `no-db` 配置：

```bash
mvn spring-boot:run -Dspring-boot.run.profiles=no-db
```

---

## 4. 验证运行是否正常

项目中提供了若干测试接口，用于验证 JPA 与数据库连接。均为 `GET` 请求，在浏览器访问即可。

| 接口 | URL | 说明 |
|------|-----|------|
| 用户列表 | `http://localhost:8080/api/test/users` | 返回所有用户（含 admin） |
| 用户行为 | `http://localhost:8080/api/test/behavior` | 创建/返回收藏与评分记录 |
| 订单与推荐 | `http://localhost:8080/api/test/order` | 创建/返回测试订单与推荐结果 |
| 装备分类 | `http://localhost:8080/api/test/gears/categories` | 返回装备分类（启动时自动初始化 20 类） |
| 装备列表 | `http://localhost:8080/api/test/gears/items` | 返回装备列表（含 450+ 真实可查装备及标签） |
| 装备汇总 | `http://localhost:8080/api/test/gears` | 返回分类、装备、标签汇总 |
| 出行计划 | `http://localhost:8080/api/test/plans` | 创建/返回用户 1 的出行计划及关联装备 |

预期行为：

- 首次启动 `local` profile 时，会自动插入：
  - 角色：`ROLE_ADMIN`、`ROLE_USER`
  - 用户：`admin`（后续接入 Spring Security 时会替换为加密密码）
  - 装备知识库：20 个分类、120+ 标签、450+ 真实可查装备及装备-标签关联
- 调用 `/api/test/users` 应返回 JSON 数组，其中包含 `admin` 用户记录。
- 调用 `/api/test/gears` 可查看分类、装备、标签的完整知识库。

如果看到以上返回，说明：

- Docker 中的 MySQL/Redis 工作正常；
- Spring Boot 数据源配置正确；
- JPA 实体与仓库读写无误。

---

## 5. 常用命令汇总

```bash
# 启动依赖服务（MySQL + Redis）
docker compose up -d

# 查看容器状态
docker ps

# 停止依赖服务
docker compose down

# 启动后端（local 配置，连 Docker MySQL/Redis）
mvn spring-boot:run -Dspring-boot.run.profiles=local

# 仅启用 Web（不连数据库/Redis）
mvn spring-boot:run -Dspring-boot.run.profiles=no-db
```

后续实现完成前端后，可在 README 中补充前端启动步骤，与本文件一起作为毕设答辩的运行说明。
