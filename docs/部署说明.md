# 部署说明

本文档说明如何打包前后端并在生产环境中运行。

---

## 1. 环境要求

- **JDK 17+**
- **Maven 3.9+**
- **Node.js 18+**（用于打包前端）
- **MySQL 8**（库名 `outdoor_gear`，已执行 `schema.sql` 及迁移脚本）
- **Redis**（可选，用于推荐结果缓存）

---

## 2. 前端打包

```bash
cd frontend
npm install
npm run build
```

产物在 `frontend/dist` 目录，可将该目录整体部署到 Nginx、对象存储或 Spring Boot 静态资源目录。

### 部署到 Nginx

- 将 `dist` 内容放到 Nginx 根目录或子路径（如 `/` 或 `/app`）。
- 若前后端分离部署，配置反向代理将 `/api` 转发到后端地址，例如：

```nginx
location /api {
    proxy_pass http://后端地址:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}
```

---

## 3. 后端打包

```bash
cd D:\outdoor-gear-recommendation
mvn clean package -DskipTests
```

生成的 JAR 位于 `target/outdoor-gear-recommendation-0.0.1-SNAPSHOT.jar`。

运行（使用生产 profile，需提前配置 `application-prod.yml` 或环境变量）：

```bash
java -jar target/outdoor-gear-recommendation-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

---

## 4. 生产环境配置

### 4.1 配置文件

建议使用 `application-prod.yml` 或环境变量覆盖敏感与环境相关配置，不要将生产密码提交到仓库。

可配置项示例：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `spring.datasource.url` | MySQL 地址、库名 | `jdbc:mysql://host:3306/outdoor_gear?...` |
| `spring.datasource.username` | 数据库用户名 | 环境变量 `MYSQL_USER` |
| `spring.datasource.password` | 数据库密码 | 环境变量 `MYSQL_PASSWORD` |
| `spring.data.redis.host` | Redis 地址 | 环境变量 `REDIS_HOST` |
| `spring.data.redis.password` | Redis 密码（若有） | 环境变量 `REDIS_PASSWORD` |
| `jwt.secret` | JWT 密钥 | 环境变量 `JWT_SECRET`，生产务必使用强随机串 |
| `server.port` | 服务端口 | 默认 8080 |
| `gear.init.force` | 是否强制初始化装备库 | 生产固定为 `false` |

主配置 `application.yml` 已使用 `${MYSQL_HOST:localhost}` 等形式，可通过环境变量覆盖。

### 4.2 数据库

1. 创建库：`CREATE DATABASE outdoor_gear ...;`
2. 执行建表：`src/main/resources/sql/schema.sql`
3. 按需执行迁移：`migration-role-apply.sql`、`migration-user-preference.sql`、`migration-plan-keywords-and-user-profile.sql`

### 4.3 CORS（前后端分离时）

当前 `SecurityConfig` 中 CORS 允许所有来源（`*`）。生产建议改为具体前端域名，例如：

```java
config.setAllowedOrigins(List.of("https://your-frontend-domain.com"));
```

## 5. 可选：Docker 部署

### 5.1 仅 MySQL + Redis（与本地开发一致）

```bash
docker compose up -d
```

后端仍在本机或其它机器运行，通过环境变量指向容器内的 MySQL/Redis。

### 5.2 后端打镜像（示例）

在项目根目录创建 `Dockerfile`（示例）：

```dockerfile
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
COPY target/outdoor-gear-recommendation-*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

构建并运行：

```bash
mvn package -DskipTests
docker build -t outdoor-gear-backend .
docker run -e MYSQL_HOST=host.docker.internal -e MYSQL_USER=... -e MYSQL_PASSWORD=... -p 8080:8080 outdoor-gear-backend
```

前端可单独用 Nginx 镜像托管 `frontend/dist`，或与后端同机用 Nginx 反代。
