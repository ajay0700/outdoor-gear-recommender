# 阶段一：开发与运行环境

> **说明**：本文档与 [快速开始.md](快速开始.md) 在「环境、Docker、启动」部分有重叠。**日常运行项目请以《快速开始》为准**；本文档保留作阶段一历史参考。

## 1. 本地环境要求

| 软件 | 版本建议 | 用途 |
|------|----------|------|
| JDK | 17+ | 后端编译与运行 |
| Maven | 3.8+ | 后端依赖与打包 |
| Node.js | 18+（LTS） | 前端开发（阶段六） |
| MySQL | 8.0+ | 主库（阶段二建表后使用） |
| Redis | 6+ | 推荐缓存等（阶段五使用，可选） |
| WSL2 + Ubuntu | 最新稳定版 | Docker 后端运行环境 |

- 未安装 MySQL/Redis 时，可先使用 **no-db** 配置启动，仅启动 Web，不连数据库与 Redis：
  ```bash
  mvn spring-boot:run -Dspring-boot.run.profiles=no-db
  ```

## 2. 数据库与配置占位

- **MySQL**：建议创建库名 `outdoor_gear`，并单独建一个账号（如 `outdoor_gear` / 密码自设），避免使用 root。
- **配置**：在 `application.yml` 中已预留占位，可通过环境变量覆盖：
  - `MYSQL_HOST`、`MYSQL_PORT`、`MYSQL_DATABASE`、`MYSQL_USER`、`MYSQL_PASSWORD`
  - `REDIS_HOST`、`REDIS_PORT`、`REDIS_PASSWORD`
- 未建库前：使用 `spring.profiles.active=no-db` 启动即可，无需注释配置。

> 说明：本机 WSL 发行版为 **Ubuntu**，已创建用于开发的普通用户 `outdoor`，主要给 Docker 等开发环境使用。

## 3. 使用 Docker 启动 MySQL + Redis（可选）

在项目根目录执行：

```bash
docker compose -f docker-compose.yml up -d
```

将启动：

- MySQL 8：端口 3306，库名 `outdoor_gear`，用户/密码见 `docker-compose.yml` 中环境变量。
- Redis：端口 6379，无密码。

连接信息与 `application.yml` 中默认占位一致时，无需改代码即可连库。

## 4. 后端启动

```bash
mvn spring-boot:run
```

或先打包再运行：

```bash
mvn -q package
java -jar target/outdoor-gear-recommendation-0.0.1-SNAPSHOT.jar
```

## 5. 前端（阶段六再创建）

- 前端项目将放在 `frontend/`（或单独仓库），使用 Vue 3 + Element Plus，通过环境变量配置后端 API 基地址。
