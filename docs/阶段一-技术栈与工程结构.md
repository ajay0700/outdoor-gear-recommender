# 阶段一：技术栈与工程结构

## 1. 技术栈确认

### 后端（与开题一致）

| 技术 | 用途 |
|------|------|
| Spring Boot 4.x | 应用框架 |
| Spring Security + JWT | 认证与授权 |
| Spring Data JPA | 持久化与仓储 |
| MySQL | 主库（用户、装备、计划、行为、社区等） |
| Redis | 推荐结果缓存、会话/Token 黑名单（可选） |
| Maven | 构建与依赖管理 |

### 前端（与开题一致）

| 技术 | 用途 |
|------|------|
| Vue 3 | 前端框架 |
| Element Plus | UI 组件库 |
| Axios | HTTP 请求 |
| Vue Router | 路由 |
| Pinia（或 Vuex） | 状态管理 |

## 2. 后端包结构（com.outdoor.gear）

```
com.outdoor.gear
├── OutdoorGearRecommendationApplication.java   # 启动类
├── config          # 全局配置（CORS、Jackson、Swagger 等）
├── controller      # 控制器（按模块：auth、user、gear、plan、recommend、cart、community、admin）
├── service         # 业务逻辑
├── repository      # JPA Repository（或 mapper 若后续引入 MyBatis）
├── entity          # JPA 实体
├── dto             # 请求/响应 DTO
├── security        # JWT 生成与校验、UserDetails 等
└── recommendation  # 推荐算法（约束、协同过滤、内容推荐及融合）
```

- **controller** 下可按子包再分：`auth`、`user`、`gear`、`plan`、`recommend`、`cart`、`community`、`admin`，便于按模块拆分。
- **entity** 与数据库表一一对应；**dto** 用于接口入参/出参，与 entity 解耦。

## 3. 前端目录规划（Vue 3 项目，后续创建）

```
frontend/                    # 或 vue-app/
├── public/
├── src/
│   ├── api/                  # 按模块：auth、user、gear、plan、recommend、cart、community、admin
│   ├── assets/
│   ├── components/           # 公共组件
│   ├── layouts/              # 布局（用户端布局、管理端布局）
│   ├── router/
│   ├── stores/               # Pinia stores
│   ├── views/
│   │   ├── auth/             # 登录、注册
│   │   ├── plan/             # 出行计划创建/编辑/列表
│   │   ├── recommend/       # 推荐结果页
│   │   ├── gear/            # 装备列表、详情、对比
│   │   ├── cart/            # 购物车、下单
│   │   ├── community/       # 论坛、帖子、评论
│   │   ├── user/            # 个人中心、收藏、订单、积分
│   │   └── admin/           # 管理端：用户、装备、审核、推荐配置、运营
│   └── App.vue / main.js
├── package.json
└── vite.config.js / vue.config.js
```
