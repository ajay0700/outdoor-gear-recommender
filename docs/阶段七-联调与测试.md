按顺序在本地验证以下流程（后端 + 前端均已启动）：

| 序号 | 流程 | 操作要点 | 预期 |
|------|------|----------|------|

### 跨域与请求配置

- **开发环境**：前端通过 Vite 代理将 `/api` 转发到 `http://localhost:8080`，同源无跨域问题。
- **后端 CORS**：`SecurityConfig` 已配置允许所有来源（`*`），便于前后端分离部署或多端口调试。生产环境建议改为具体前端域名。

### Token 过期与 403

| 现象 | 原因 | 处理 |
|------|------|------|
| 请求返回 401 | Token 过期或未携带 | 前端清除本地 token，跳转登录页重新登录 |
| 请求返回 403 | 无权限（如普通用户访问管理端接口） | 检查角色；管理端接口需 ADMIN/GEAR_ADMIN/EXPERT 等角色 |
| 登录后仍 401 | 请求头未带 `Authorization: Bearer <token>` | 检查前端 request 拦截器是否在 header 中注入 token |

前端已在 `src/api/request.ts` 中统一添加 Token；Token 过期时可根据响应 401 清除存储并跳转登录。
