# 阶段二：数据库设计

本文档描述户外运动装备推荐系统的数据库概念模型与表结构，对应阶段一需求与功能清单。

---

## 1. 设计约定

- **主键**：各表统一使用 `id BIGINT AUTO_INCREMENT PRIMARY KEY`。
- **时间**：创建/更新时间字段 `created_at`、`updated_at` 使用 `DATETIME`。
- **逻辑删除**：业务表（用户、装备等）使用 `is_deleted TINYINT(1) DEFAULT 0`（0-未删除，1-已删除）。
- **字符集**：库与表使用 `utf8mb4`，排序规则 `utf8mb4_unicode_ci`。
- **存储引擎**：InnoDB。

## 8. 与推荐算法的对应关系

- **约束过滤**：依赖 `trip_plan`（季节、预算、活动类型、人数等）与 `gear_item`（season、scene、price、category_id 等）做多维筛选。
- **协同过滤**：依赖 `user_favorite`、`user_cart_item`、`gear_rating`、`gear_browse_log`、`order_item` 等行为数据构建用户-物品矩阵。
- **内容推荐**：依赖 `gear_item` 属性与 `gear_tag`/`gear_item_tag` 做特征相似度。
- **推荐结果**：可写入 `recommend_result` 做效果统计与论文实验。

## 9. 建表脚本与初始化数据

- **建表脚本**：`src/main/resources/sql/schema.sql`
- **装备知识库初始化**：
  - Java：`GearKnowledgeInitializer`（`@Profile("local")`）在应用启动时自动初始化分类、标签、示例装备及装备-标签关联
  - SQL 备用：`src/main/resources/sql/data-init-gear.sql` 可手动执行分类与标签初始化
- 首次建库后，建议将 JPA 的 `ddl-auto` 设为 `validate`，由脚本与实体共同维护表结构。
